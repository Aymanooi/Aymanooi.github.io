# عقد التنفيذ النهائي (قبل الكود) — منع الانحراف

> آخر تثبيت قبل أمر التنفيذ. كل بند قرار قاطع لا يتغيّر أثناء البناء. **لا كود في هذا المستند.**

---

## 1) واجهة الإدخال/الإخراج في `lib/converters.js`

**القرار: طبقتان فقط — أساس عام + غلاف مُسمّى. قابل للتوسّع بلا over-engineering.**

| الدالة | التوقيع | السلوك |
|---|---|---|
| `convertLinear(value, factor)` | `(number, number) → number` | ترجع `value * factor` خامًا (دقة كاملة). هذا **مفصل التوسّع**: الصفحة 2 (كجم→رطل) تعيد استخدامها بـ`factor` آخر |
| `metersToFeet(value)` | `(number) → number` | غلاف رفيع = `convertLinear(value, 3.28084)` |

**لماذا هذا "قابل للتوسّع بلا إفراط":**
- أساس عام واحد (`convertLinear`) + اسم واضح واحد (`metersToFeet`). **لا سجلّ وحدات، لا كائنات إعداد، لا factory.** فقط ما يكفي ليُعاد استخدام الأساس لاحقًا.
- **مدخل غير صالح:** `value` يُمرَّر عبر `Number()`؛ إن كان الناتج `غير منتهٍ` (NaN/Infinity) → الدالة ترجع `NaN` (سلوك معرّف ومُختبَر).
- **الإخراج خام (raw float).** التقريب/التنسيق **ليس** مسؤولية المنطق — يحدث في طبقة العرض. هذا يبقي المنطق نقيًا قابلًا للاختبار.

## 2) اختبار النجاح: المقارنة والإبلاغ

**القرار: مقارنة بـ tolerance (ليست strict equality)، عبر `assert.ok` مع مساعد محلي صغير داخل ملف الاختبار فقط.**

- **لماذا ليست `===`:** نتعامل مع أعداد عائمة (floats)؛ المساواة الصارمة هشّة. نستخدم:
  `Math.abs(actual - expected) < 1e-4`.
- **الآلية:** مساعد محلي `assertClose(actual, expected, tol = 1e-4, msg)` **داخل ملف الاختبار حصرًا** (ليس مكتبة، ليس إطارًا) يلفّ `assert.ok(...)` من `node:assert/strict`.
- **حالات الاختبار:**
  | الإدخال | المتوقّع |
  |---|---|
  | `metersToFeet(1)` | `≈ 3.28084` |
  | `metersToFeet(0)` | `0` |
  | `metersToFeet(2.5)` | `≈ 8.2021` |
  | `metersToFeet(-3)` | `≈ -9.84252` |
  | `metersToFeet("abc")` | `NaN` (عبر `Number.isNaN`) |
  | `convertLinear(10, 2)` | `20` (الأساس العام) |
- **أين يظهر الفشل عمليًا:** في **الطرفية / CI** عبر `node --test` — مخرجات TAP تطبع `not ok` مع رسالة `assertClose` (المتوقّع مقابل الفعلي)، ورمز خروج غير صفري. **لا يظهر في المتصفح** — الاختبار طبقة بناء لا طبقة تشغيل.

## 3) تدفّق execution داخل المتصفح (تسلسل فعلي)

**القرار: استيراد وحدة واحدة → حساب أوّلي عند التحميل → تحديث بالطفرة المباشرة (direct mutation)، بلا re-render.**

```
1. المتصفح يحلّل HTML من الأعلى. <link rel="stylesheet"> في <head> يُطبَّق.
2. في نهاية <body>: <script type="module"> — الوحدات مؤجَّلة افتراضيًا (defer)،
   تُنفَّذ بعد اكتمال تحليل DOM.
3. الاستيراد أولًا:  import { metersToFeet } from "../../lib/converters.js"
   → المتصفح يجلب ملف الوحدة النقي (ملف ثابت واحد).
4. عند تنفيذ الوحدة:
   - الحصول على المراجع مرة واحدة:
       input  = document.getElementById("value")
       output = document.getElementById("result")
   - تعريف update():
       n = Number(input.value)
       r = metersToFeet(n)
       output.textContent = Number.isNaN(r) ? "—" : r.toFixed(4) + " قدم"
5. حساب أوّلي:  update()  ← يُستدعى مرة عند التحميل (الحقل مُعبّأ بـ 1 → يعرض 3.2808 قدم)
6. ربط الحدث:  input.addEventListener("input", update)
7. عند كتابة المستخدم: حدث input → update() →
   **طفرة مباشرة لعقدة نصّية واحدة** (output.textContent = ...).
   لا re-render، لا innerHTML، لا virtual DOM، لا إعادة بناء للصفحة.
```

- **re-render أم direct mutation؟** → **direct mutation** لعقدة نصّ واحدة. أرخص وأسرع تحديث ممكن.
- **التنسيق:** `toFixed(4)` في طبقة العرض فقط (المنطق يبقى خامًا).
- **صفر طلبات شبكة** بعد تحميل الصفحة؛ تعمل بلا اتصال.

---

## 4) منع الانحراف: لا abstraction إضافي في الواجهة

**القرار: مفروض وصارم لهذه المرحلة.**

| ممنوع الآن | مسموح |
|---|---|
| ❌ أي framework (React/Vue/...) | ✅ DOM أصلي (vanilla) فقط |
| ❌ أي خطوة بناء/bundler | ✅ ES module مستورد مباشرة |
| ❌ ملف `helpers.js` أو طبقة أدوات | ✅ المنطق النقي في `lib/` + غراء مضمّن في الصفحة |
| ❌ أي اعتماد خارجي (npm runtime) | ✅ المتصفح + `node:test` للاختبار فقط |
| ❌ تعميم مبكّر (سجلّ وحدات، إعدادات) | ✅ `convertLinear` + `metersToFeet` فقط |

> الاستيراد الوحيد في الصفحة هو الدالة النقية. السكربت المضمّن **غراء يربط الإدخال بالمنطق بالـDOM** — لا أكثر.

## 5) تعريف "تم النجاح" — جملة واحدة قابلة للتحقّق

> **عند فتح `tools/meter-to-foot/index.html`، تعرض الصفحة فورًا «1 متر = 3.2808 قدم» وتُحدّث الناتج صحيحًا لأي إدخال رقمي، و`node --test` يمر دون فشل — والصحّة الحسابية هي المقياس الوحيد لهذه المرحلة.**

(زمن التحميل الفوري خاصيةٌ بنيوية لصفحة ثابتة، لا بوّابة مُقاسة الآن — تبقى مُجمّدة كمقياس وفق نقطة التجميد.)

## 6) ترتيب التنفيذ عند الـ go
```
1) lib/converters.js          → convertLinear + metersToFeet
2) tests/converters.test.js   → assertClose + الحالات الستّ → node --test يمر
3) tools/meter-to-foot/index.html → الصفحة + غراء الاستيراد
4) تحقّق يدوي في المتصفح + commit/push للفرع
```

> هذا العقد مُقفل. أي تغيير عليه يتطلّب قرارًا صريحًا، لا اجتهادًا أثناء الكتابة.
