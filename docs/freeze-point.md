# نقطة التجميد (Freeze Point): الشريحة العمودية الوحيدة

> قرار هندسي واحد صارم يكسر شلل التحليل. **تعريف فقط — لا كود حتى قرار الـ go.**
> الهدف: أصغر شيء **حقيقي وقابل للشحن** يثبت النواة، لا "مفهوم صفحة".

---

## 1) مقياس النجاح الوحيد (اختيار واحد فقط)

> ✅ **صحّة الحساب فقط (Calculation Correctness).**

**لماذا هذا دون غيره:**
- محوّل سريع ومفهرس لكنه يعطي رقمًا خاطئًا = **عديم القيمة بل ضار**. الصحّة هي النواة غير القابلة للاختزال.
- الوحيد **القابل للقياس فورًا وحتميًا** الآن (اختبار حتمي). السرعة/الفهرسة/الاستخدام إمّا تابعة أو تتأخّر أسابيع.
- البقية (سرعة، فهرسة، استخدام) **مُجمّدة كمقاييس** لهذه المرحلة — تُقاس لاحقًا، لا الآن.

**التعريف الإجرائي للنجاح:** `metersToFeet(1) === 3.28084 (±1e-4)` وكل حالات الاختبار تمر. لا شيء غير ذلك يحدّد نجاح هذه المرحلة.

## 2) الحد الأدنى المطلق — الملفات (3 ملفات جديدة + إعادة استخدام CSS)

| الملف | الحالة | الوظيفة الدقيقة |
|---|---|---|
| `lib/converters.js` | جديد | **المنطق النقي فقط.** يصدّر `metersToFeet(value)` (و`convertLinear(value, factor)` كأساس). بلا DOM، بلا أثر جانبي. هذا قلب "الصحّة" |
| `tests/converters.test.js` | جديد | **مقياس النجاح يعيش هنا.** يؤكّد `1→3.28084`, `0→0`, قيمة سالبة, قيمة عشرية. `node --test` يجب أن يمر |
| `tools/meter-to-foot/index.html` | جديد | **الصفحة الفعلية.** واجهة + SEO + محتوى. JS مضمّن صغير يستورد `lib/converters.js` ويربطه بالـ DOM فقط (لا حساب في HTML) |
| `assets/styles.css` | موجود ✅ | يُعاد استخدامه كما هو. لا ملف تنسيق جديد |

**لا شيء غير هذه الأربعة.** لا `data/`, لا `scripts/`, لا `.github/`, لا `app.js`.

## 4) تعريف التنفيذ الدقيق لصفحة `meter → foot`

### 4.1 ماذا يفعل كل ملف
- **`lib/converters.js`** — دالة `convertLinear(value, factor)` ترجع `value × factor`؛ و`metersToFeet(value)` تستدعيها بـ`factor = 3.28084`. نقية ومُختبَرة.
- **`tests/converters.test.js`** — حالات: `1→3.28084`، `0→0`، `2.5→8.2021`، قيمة سالبة، إدخال غير رقمي → سلوك معرّف. **هذا هو مقياس النجاح.**
- **`tools/meter-to-foot/index.html`** — يحتوي:
  - `<head>`: `lang="ar" dir="rtl"`، `<title>` فريد، `meta description`، `canonical`، خط Cairo، رابط `../../assets/styles.css`، JSON-LD (`WebApplication` + `FAQPage`) مكتوب لهذه الصفحة.
  - `<body>`: شريط بسيط، `<h1>تحويل المتر إلى قدم</h1>`، **لوحة الأداة** (حقل رقمي + نتيجة حيّة)، **جدول قيم مرجعي** (1،2،5،10،50،100 متر)، **كتلة محتوى** (كيف تستخدم؟ • حالات • FAQ)، تذييل.
  - `<script type="module">`: `import { metersToFeet } from "../../lib/converters.js"` ثم ربط حدث `input` → حساب → عرض. **غراء فقط، صفر منطق.**

### 4.2 ماذا يحدث عند فتح الصفحة (خطوة بخطوة)
1. المتصفح يحمّل `index.html` الثابت → **فوري** (لا خادم).
2. `assets/styles.css` يُطبَّق (مُخزّن) → عرض RTL أنيق.
3. الصفحة تُرسَم: العنوان، حقل مُعبّأ بـ`1`، والنتيجة تعرض `3.28084 قدم` (محسوبة عند التحميل).
4. وحدة JS تُحمّل، تستورد الدالة النقية، تربط مستمع `input`.
5. المستخدم يكتب رقمًا (مثلًا `10`) → المستمع يستدعي `metersToFeet(10)` → يحدّث النص إلى `32.8084 قدم` في **< 100ms، بلا أي طلب شبكة**.
6. تعمل **بلا اتصال** بعد التحميل الأول. **صفر طلبات خادم.**

### 4.3 النتيجة القابلة للقياس
- فتح الصفحة يُظهر تحويلًا صحيحًا؛ كتابة `N` تُظهر `N × 3.28084` بأربع منازل.
- `node --test` يمر (مقياس النجاح الوحيد).

## 5) القرار الحاسم: هل نبدأ الآن أم نبسّط أكثر؟

> **القرار: نبدأ الآن. لا حاجة لتبسيط إضافي.**

**التبرير:** الشريحة بلغت **الحد غير القابل للاختزال**:
- لا يمكن إسقاط الاختبار = هو المقياس.
- لا يمكن إسقاط `lib` = هو المنطق.
- لا يمكن إسقاط الصفحة = هي المُخرَج.
أي تبسيط إضافي يحذف جوهرًا، لا زيادة. هذه **أنحف نسخة حقيقية ممكنة**.

**ترتيب البناء عند الـ go:**
```
1) lib/converters.js        (المنطق)
2) tests/converters.test.js (المقياس) → شغّل node --test → يجب أن يمر
3) tools/meter-to-foot/index.html (الصفحة) → افتحها → تحقّق يدويًا من السلوك
4) commit + push للفرع
```

> بعد نجاح هذه الشريحة وفهرستها لاحقًا، نفكّ تجميد أوّل بند NICE-TO-HAVE فقط عند الحاجة (الصفحة الثانية → ثم المولّد عند الصفحة 3–5).
