# مخطط المشروع (Blueprint): الأهداف + المعمارية + أول 10 أدوات + القالب

> **الحالة:** تصميم بانتظار الاعتماد (Design — pending lock).
> **القاعدة الصارمة:** لا يبدأ أي كود لأداة جديدة قبل اعتماد هذا المستند (architecture + 10 tools + template).
> **الهدف المعماري:** أساس يتوسّع إلى **100+ أداة دون إعادة هيكلة**.

## الجزء 0 — إعادة صياغة الهدف بشكل قابل للقياس

### تعريف النجاح خلال 30 يومًا (مؤشرات قائدة، لا إيرادات)

SEO لا يثمر إيرادًا خلال شهر؛ لذلك نقيس **المدخلات والجودة**، لا المال:

| # | المؤشر | الهدف خلال 30 يومًا | لماذا يقاس |
|---|---|---|---|
| 1 | عدد الأدوات المنشورة والعاملة | **10 أدوات** | الكتلة الحرجة للبدء |
| 2 | الفهرسة في Google | **10/10 صفحات مفهرسة** | شرط ظهور أي حركة |
| 3 | Core Web Vitals (Lighthouse) | **≥ 95 أداء + ≥ 95 SEO** لكل صفحة | الأداء عامل ترتيب وتجربة |
| 4 | ظهور بحثي (Impressions) في Search Console | **> 0 وبدأ ينمو** | إشارة أن الكلمات تُلتقط |
| 5 | معدل اكتمال المهمة بلا أخطاء | **100% client-side، صفر طلبات خادم** | جوهر نموذج "تكلفة صفر" |
| 6 | تغطية الاختبارات للمنطق النقي | **كل دالة منطق لها اختبار يمر** | منع الانحدار مع التوسّع |

> الإيراد **ليس** هدف الـ30 يومًا. الهدف هو أساس مفهرَس وسريع وصحيح. الإيراد دالة على الحركة، والحركة دالة على الوقت + الجودة.

### الحد الأدنى من القيمة لأول 10 أدوات (تعريف "Done")

كل أداة **يجب** أن تحقق هذه الشروط وإلا لا تُنشر:

1. **تنجز المهمة كاملة في المتصفح** — بلا تسجيل، بلا رفع لخادم، بلا انتظار.
2. **نتيجة فورية** (< 100ms للأدوات النصية/الحسابية؛ معالجة محلية للصور).
3. **خصوصية مطلقة** — بيانات المستخدم لا تغادر جهازه (نقطة بيع أساسية ومصدر ثقة).
4. **أفضل من البديل العربي المتوسط** — أنظف، أسرع، بلا إعلانات مزعجة، بعربية صحيحة.
5. **تعمل بلا اتصال** بعد التحميل الأول (قابلة لـ PWA لاحقًا).
6. **منطقها معزول في `/lib` ومُختبَر**.

### فرضية النمو الأساسية

نرتّب القنوات حسب الأدلة على قيود مشروعنا (فردي، ميزانية صفر):

| الأولوية | القناة | الفرضية | لمن تصلح |
|---|---|---|---|
| **1 (الأساسية)** | **SEO طويل الذيل (Long-tail)** | كلمات بحث عربية محددة وعالية النية ("محوّل التاريخ الهجري"، "تفقيط المبلغ") بمنافسة أقل من الإنجليزية → ترتيب أعلى → حركة مجانية متراكمة | كل الأدوات |
| 2 (مضاعِف) | **قابلية المشاركة (Virality)** | أدوات تنتج مخرجًا قابلًا للمشاركة (QR، "كم عمرك"، صورة مضغوطة) تُشارك طبيعيًا | أدوات مختارة |
| 3 (شبكي) | **الربط الداخلي بين الأدوات** | كل صفحة أداة تقترح أدوات ذات صلة → تزيد الصفحات/الجلسة وسلطة النطاق | كل الأدوات |

> **الرهان الأساسي = SEO.** الميزة التنافسية الحاسمة = **الفجوة في الأدوات العربية عالية الجودة**، خصوصًا الأدوات ذات الطابع العربي الأصيل (هجري، تفقيط) حيث المنافسة أضعف والقيمة أوضح.

## الجزء 1 — معمارية الموقع (Site Architecture)

### 1.1 تنظيم الأدوات (التصنيفات)

الأدوات مجمّعة في **فئات** تظهر في الصفحة الرئيسية وفي مسار التنقّل (breadcrumb):

```
نص (Text)        │ عدّاد كلمات، تحويل حالة، تفقيط، Lorem عربي ...
صور (Images)     │ ضغط الصور، تحويل الصيغ، تغيير الأبعاد ...
مطوّرون (Dev)    │ منسّق JSON، Base64، تشفير URL ...
تاريخ (Date)     │ محوّل هجري/ميلادي، حاسبة العمر، الفرق بين تاريخين ...
حساب (Calc)      │ النسبة المئوية، الوحدات، الضريبة ...
أمان (Security)  │ مولّد كلمات مرور، مولّد QR ...
```

التصنيف **بيانات وليس كودًا**: معرّف في `data/tools.js` (مصدر الحقيقة الوحيد).

### 1.2 نظام الـ Routing — صفحة مستقلة لكل أداة

قرار حاسم لـ SEO: **كل أداة = صفحة HTML مستقلة بمسار نظيف**.

```
/                                  → الصفحة الرئيسية (hub: كل الأدوات بالفئات)
/tools/word-counter/               → أداة عدّاد الكلمات
/tools/image-compressor/           → أداة ضغط الصور
/tools/hijri-converter/            → محوّل التاريخ
/category/text/                    → (لاحقًا) صفحة فئة "نص"
/sitemap.xml                       → خريطة الموقع لمحركات البحث
/robots.txt                        → توجيه الزواحف
```

- **لماذا مجلد لكل أداة (`/tools/word-counter/index.html`) وليس `word-counter.html`؟**
  → ينتج URL نظيفًا `/tools/word-counter/` (بلا امتداد `.html`)، وهو معيار SEO أفضل وأسهل للمشاركة.
- **لا توجد توجيهات JavaScript (SPA routing).** كل صفحة HTML حقيقية مستقلة → فهرسة فورية وموثوقة، وأداء أعلى، وعمل بلا JS.
- **التوسّع لـ100+ أداة:** إضافة أداة = مجلد جديد + إدخال واحد في `data/tools.js`. لا لمس للبنية.

### 1.4 استراتيجية SEO لكل صفحة (قالب موحّد لكل أداة)

كل صفحة أداة تلتزم بهذه العناصر إلزاميًا (مولّدة جزئيًا من `tools.js`):

1. **`<title>` فريد** بصيغة: `«اسم الأداة» — أداة مجانية أونلاين | أدوات`.
2. **`<meta name="description">` فريد** (140–160 حرفًا، يتضمن الكلمة المفتاحية والفائدة).
3. **`<link rel="canonical">`** للمسار النظيف.
4. **Open Graph + Twitter Card** للمشاركة الاجتماعية.
5. **`lang="ar" dir="rtl"`** على مستوى الصفحة.
6. **بيانات منظّمة JSON-LD:**
   - `WebApplication` (نوع الأداة، السعر = 0).
   - `BreadcrumbList` (الرئيسية ← الفئة ← الأداة).
   - `FAQPage` (أسئلة شائعة حقيقية لكل أداة — تلتقط مقتطفات مميّزة في Google).
7. **كتلة محتوى نصية (≥ 250 كلمة)** أسفل الأداة: "كيف تستخدم؟"، "حالات الاستخدام"، "لماذا أداتنا؟"، FAQ. (الأدوات الخالية من نص لا تترتّب.)
8. **ربط داخلي:** قسم "أدوات ذات صلة" (3–4 أدوات من نفس الفئة).
9. **عنوان H1 واحد** مطابق لنية البحث.
10. **أداء:** بلا إطار عمل، CSS مشترك مُخزّن (cache)، JS الأداة مقسّم لكل صفحة فقط.

## الجزء 2 — أول 10 أدوات (مع المبررات)

اختيار مبني على: **طلب بحث مرتفع + قابلية تنفيذ 100% client-side + فجوة عربية + توزيع على الفئات**.

| # | الأداة | الفئة | سبب الاختيار (الطلب/القيمة) | المنافسة | قابلية الانتشار | الميزة |
|---|---|---|---|---|---|---|
| 1 | **عدّاد الكلمات والأحرف** | نص | طلب مرتفع (طلاب/كتّاب/سوشيال)، استخدام متكرر | متوسطة | منخفضة | المنطق جاهز ومُختبَر |
| 2 | **منسّق ومدقّق JSON** | مطوّرون | طلب مرتفع بين المطوّرين، استخدام يومي | عالية | منخفضة | فجوة عربية للمطوّرين |
| 3 | **ضغط الصور** | صور | طلب ضخم ("ضغط صورة")، قيمة واضحة فورية | عالية | منخفضة | خصوصية كاملة (محلي) |
| 4 | **تحويل صيغ الصور** (PNG/JPG/WebP) | صور | طلب مرتفع، Canvas بحت بلا خادم | متوسطة | منخفضة | سريع ومحلي |
| 5 | **مولّد كلمات مرور قوية** | أمان | طلب مرتفع، نية أمان عالية | عالية | منخفضة | "لا يغادر جهازك" = ثقة |
| 6 | **مولّد QR Code** | أمان/أدوات | طلب مرتفع، مخرج قابل للمشاركة | عالية | **متوسطة-عالية** | QR يُشارك = نمو فيروسي |
| 7 | **محوّل التاريخ الهجري ↔ الميلادي** | تاريخ | طلب عربي **ضخم وخاص**، منافسة عالمية ضعيفة | **منخفضة** | متوسطة | 🏆 ميزة عربية أصيلة |
| 8 | **تفقيط: الأرقام إلى كلمات عربية** | نص | يُستخدم في الفواتير/الشيكات (B2B متكرر) | **منخفضة** | منخفضة | 🏆 فجوة عربية واضحة |
| 9 | **حاسبة النسبة المئوية** | حساب | طلب مرتفع وعام، بسيطة وسريعة | متوسطة | منخفضة | جودة + عربية نظيفة |
| 10 | **حاسبة العمر / الفرق بين تاريخين** | تاريخ | طلب مرتفع ("كم عمري")، مخرج شخصي | متوسطة | **متوسطة** | قابلة للمشاركة |

### منطق المحفظة (Portfolio logic)
- **3 أدوات منطقها جاهز ومُختبَر** (1، 2، 3) → إطلاق سريع.
- **أداتان بميزة عربية أصيلة ومنافسة منخفضة** (7، 8) → أعلى احتمال ترتيب مبكّر = "رأس الحربة" في SEO.
- **أداتان قابلتان للمشاركة** (6، 10) → بذرة النمو الفيروسي.
- **توزيع متوازن** عبر 6 فئات → يثبت أن المعمارية تتسع لكل الأنواع دون إعادة هيكلة.

### اعتمادات خارجية (بحد أدنى صارم)
- 8 أدوات: **صفر اعتمادات** (JS أصلي + Canvas + Intl).
- QR (6): مكتبة صغيرة جدًا client-side فقط (تُحمّل في صفحتها فقط).
- الهجري (7): عبر `Intl.DateTimeFormat` بتقويم `islamic-umalqura` (مدمج في المتصفح، صفر اعتماد).

## الجزء 3 — القالب الموحّد للأداة الواحدة (Tool Template)

كل أداة تتبع هذا العقد بالضبط (يضمن التوسّع لـ100+ بلا إعادة هيكلة):

### 3.1 الهيكل (نفس ترتيب كل صفحة)

```
┌─ <head>: meta فريدة + JSON-LD + canonical + روابط /assets المشتركة
├─ شريط تنقّل مشترك (يُحقن من app.js)         ← موحّد
├─ مسار breadcrumb: الرئيسية ← الفئة ← الأداة
├─ <header> الأداة: H1 + جملة فائدة واحدة
├─ ┌──────────── لوحة الأداة (التفاعل) ────────────┐
│  │  إدخال بسيط (textarea / dropzone / حقول)        │  ← UI أدنى
│  │  ضوابط (أزرار/منزلقات)                          │
│  │  نتيجة فورية + إحصائيات/معاينة                   │
│  │  زر نسخ/تنزيل + رسالة حالة                       │
│  └────────────────────────────────────────────────┘
├─ كتلة SEO: كيف تستخدم؟ • حالات الاستخدام • لماذا؟ • FAQ (≥250 كلمة)
├─ "أدوات ذات صلة" (ربط داخلي، 3–4)               ← من tools.js
└─ تذييل مشترك (يُحقن من app.js)                   ← موحّد
```

### 3.2 المبادئ الإلزامية

| المبدأ | التطبيق |
|---|---|
| **UI بسيط** | إدخال واحد واضح، نتيجة فورية، أقل عدد أزرار ممكن، بلا تشتيت |
| **بلا backend** | كل المعالجة في المتصفح؛ صفر طلبات شبكة بعد تحميل الصفحة |
| **أداء فائق** | بلا إطار عمل؛ CSS/JS مشترك مُخزّن؛ JS الأداة فقط في صفحتها؛ خطوط بـ `display=swap` |
| **العربية أولًا** | `dir="rtl"`، خط Cairo، نصوص ومصطلحات عربية صحيحة، أرقام حسب السياق |
| **الخصوصية** | تأكيد مرئي: "تعمل بالكامل في متصفحك — بياناتك لا تغادر جهازك" |
| **الوصولية (a11y)** | تباين كافٍ، تسميات `label`، تنقّل لوحة مفاتيح، `aria` للحالة |
| **منطق معزول** | كل حساب في `/lib` (مُختبَر)؛ ملف الأداة يربط المنطق بالـ DOM فقط |

### 3.3 عقد البيانات `data/tools.js` (نموذج الإدخال)

كل أداة = كائن بهذا الشكل (يقود الـ hub + الربط الداخلي + sitemap):

```js
{
  slug: "word-counter",
  title: "عدّاد الكلمات والأحرف",
  icon: "📝",
  category: "text",
  short: "احسب عدد الكلمات والأحرف والجُمل فورًا.",
  description: "وصف SEO فريد 140-160 حرفًا ...",
  keywords: ["عداد كلمات", "حساب الاحرف", "عدد الكلمات"],
  faq: [{ q: "...", a: "..." }],
  related: ["case-converter", "lorem-arabic", "tafqit"]
}
```

> إضافة أداة = إضافة كائن هنا + مجلد `tools/<slug>/index.html` يتبع القالب. **لا تعديل بنيوي.**

## الجزء 4 — البوابة الصارمة قبل البرمجة (Definition of Ready)

**لن يُكتب أي كود لأداة قبل تحقّق كل ما يلي:**

- [ ] **المعمارية معتمدة** (routing بمجلد لكل أداة + بنية المجلدات + SEO لكل صفحة).
- [ ] **أول 10 أدوات معتمدة** (القائمة أعلاه).
- [ ] **القالب الموحّد معتمد** (الهيكل + المبادئ + عقد `tools.js`).

عند الاعتماد، يبدأ التنفيذ بهذا الترتيب: `data/tools.js` → القالب المرجعي (أداة واحدة كاملة كنموذج) → بقية الأدوات → `sitemap` + اختبارات → نشر.

> **ملاحظة:** الأساس المُنجز سابقًا (`/lib` المُختبَر + `assets/styles.css`) متوافق مع هذا التصميم ويُعاد استخدامه؛ ملف `index.html` القديم (نسخة Portfolio) سيُستبدل بالـ hub المبني من `tools.js` في مرحلة التنفيذ.
