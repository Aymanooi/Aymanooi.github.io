# كاشف مراجحة القروض الوميضية على سولانا — وضع المحاكاة فقط

أداة **تعليمية وبحثية** تكتشف فرص المراجحة الثلاثية (triangular arbitrage)
على شبكة سولانا، وتحسب **صافي الربحية** لقرض وميضي (flash loan) بعد خصم كل
الرسوم — **دون تنفيذ أي معاملة فعلية**.

## ⚠️ نطاق الأداة (مهم)

- ✅ تقرأ أسعاراً حقيقية من **Jupiter Quote API** (نقطة `/quote` فقط).
- ✅ تحسب الربح/الخسارة بعد رسوم القرض الوميضي + رسوم السلسلة + الأولوية.
- ❌ **لا** تبني ولا توقّع ولا ترسل أي معاملة.
- ❌ **لا** تتعامل مع أي مفتاح خاص أو محفظة.
- 🎯 الهدف: فهم آلية المراجحة والقروض الوميضية على سولانا بأمان كامل.

> الاستخدامات الشرعية للقروض الوميضية: المراجحة، تصفية المراكز، وتبديل
> الضمانات. هذه الأداة لا تساعد على استغلال ثغرات البروتوكولات.

## ما هو القرض الوميضي؟

اقتراض مبلغ **بدون ضمان** بشرط إعادته داخل **نفس المعاملة الذرّية**. إن لم
يُسدَّد بنهاية المعاملة، تُلغى المعاملة بالكامل. على سولانا يتم ذلك عبر
بروتوكولات مثل **Solend / Kamino / MarginFi** التي توفّر تعليمتي
`flashBorrow` و `flashRepay`.

## كيف يعمل الكاشف؟

نبدأ بمبلغ X من عملة أساسية (مثلاً USDC)، ونمرّ عبر حلقة تبديلات:

```
USDC → SOL → BONK → USDC
```

كل قفزة تستخدم أفضل مسار من Jupiter. ثم نقارن المبلغ النهائي بالابتدائي
بعد خصم:

| الرسوم | المصدر |
|--------|--------|
| رسوم القرض الوميضي | نسبة من المبلغ المقترض (`flash_loan_fee_bps`) |
| رسوم المعاملة الأساسية | ثابتة على سولانا (~5000 lamports) |
| رسوم الأولوية | تتغيّر حسب الازدحام |
| إكرامية Jito | اختيارية (للمنافسة الحقيقية) |

إن كان: `الناتج النهائي > الابتدائي + الرسوم` ⇐ **فرصة مراجحة نظرية**.

## التشغيل

لا تبعيات خارجية — Python 3.10+ فقط:

```bash
python3 main.py                      # فحص واحد بالإعدادات الافتراضية
python3 main.py --amount 50000       # مبلغ بدء مختلف
python3 main.py --base USDC --show-all   # إظهار كل الحلقات حتى الخاسرة
python3 main.py --flash-fee-bps 5    # افتراض رسوم قرض أقل (مثل Kamino)
python3 main.py --watch 15           # تكرار كل 15 ثانية
python3 main.py --csv results.csv    # تسجيل كل النتائج في ملف CSV
```

### تسجيل النتائج في CSV

أضف `--csv <path>` لحفظ نتيجة كل حلقة في كل فحص. الملف يحوي الطابع الزمني،
الحلقة، الربح الإجمالي/الصافي، الرسوم، التأثير السعري، وهل كانت فرصة — مناسب
للتحليل لاحقاً (Excel / pandas) أو رصد الفرص عبر الزمن مع `--watch`.

### محاكاة التنفيذ على نمط devnet

```bash
python3 devnet_sim.py --amount 10000                    # محاكاة الحلقة الافتراضية
python3 devnet_sim.py --cycle USDC SOL JUP USDC         # حلقة مخصّصة
python3 devnet_sim.py --check-devnet                    # + فحص اتصال devnet RPC
python3 devnet_sim.py --flash-fee-bps 5 --amount 50000  # إعدادات مخصّصة
```

`devnet_sim.py` يبني **تسلسل التعليمات الفعلي** لقرض وميضي:

```
flashBorrow → swap₁ → swap₂ → … → flashRepay
```

ثم يطبّقه على **دفتر أستاذ وهمي محلي** ويفرض **قاعدة الذرّية**: إن لم يكفِ
الناتج لسداد القرض + الرسوم، تُلغى المعاملة بالكامل (rollback) — تماماً كما
يحدث على سولانا. النِسَب حقيقية من Jupiter، لكن **الأموال وهمية بالكامل**
ولا تُرسل أي معاملة حقيقية ولا يُستخدم أي مفتاح خاص. خيار `--check-devnet`
يتصل بـ devnet RPC للقراءة فقط (health + slot) لإثبات الاتصال.

## مثال على المخرجات

```
✅ فرصة  USDC → SOL → BONK → USDC
   البدء: 10,000.00 USDC  →  النهاية: 10,045.20 USDC
   ربح إجمالي: 45.20 USDC | رسوم: 30.04 USDC
   صافي الربح: +15.16 USDC (+0.1516%) | أقصى تأثير سعري: 0.18%
```

> ملاحظة واقعية: في معظم الأوقات لن تجد فرصاً مربحة، لأن السوق فعّال
> والبوتات الاحترافية (بـ Jito و RPC خاص) تلتقطها خلال أجزاء من الثانية.
> هذا بالضبط ما تُظهره الأداة: لماذا المراجحة الفردية صعبة عملياً.

## بنية المشروع

| الملف | الوظيفة |
|-------|---------|
| `config.py` | الإعدادات: العملات (15+)، الرسوم، الحلقات (13) |
| `jupiter.py` | عميل قراءة فقط لواجهة Jupiter Quote |
| `detector.py` | منطق تقييم الحلقات وحساب الربحية |
| `logger.py` | تسجيل النتائج في CSV |
| `main.py` | واجهة سطر الأوامر والعرض للكاشف |
| `devnet_sim.py` | محاكاة التنفيذ الذرّي على نمط devnet (أموال وهمية) |

## خطوات لاحقة مقترحة (آمنة)

1. نمذجة أدق لرسوم الأولوية بناءً على بيانات الازدحام الحقيقية.
2. تحليل ملفات CSV المتراكمة (pandas) لرصد أنماط الفرص.
3. إضافة بروتوكولات مرجعية للرسوم (Solend/Kamino/MarginFi) كإعدادات جاهزة.
4. (متقدّم جداً) ربط `devnet_sim.py` ببرنامج فعلي على devnet عبر Anchor —
   يتطلّب Rust ونشر برنامج على شبكة الاختبار، ويبقى بأموال وهمية.
