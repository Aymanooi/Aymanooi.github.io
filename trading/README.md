# ❄️ Snowball Paper — نظام تداول وهمي (Paper Trading)

نسخة نظيفة ومُصحَّحة من `snowball_v22_FINAL.py` الأصلي (6890 سطراً)، مُعاد بناؤها
من الصفر في ملف واحد قابل للتدقيق (~600 سطر).

> ⚠️ **هذا النظام تداول وهمي 100%.** لا يحتوي على أي كود لتنفيذ أوامر حقيقية،
> ولا يقبل مفاتيح API، ولا يلمس أي أموال. يستخدم أسعار OKX الحية الحقيقية
> (API عام للقراءة فقط) حتى تكون النتائج صادقة مع السوق الفعلي.

## لماذا أُعيدت كتابته؟

فحص النسخة الأصلية كشف الآتي:

| المشكلة في الأصلية | الإصلاح هنا |
|---|---|
| **لا يوجد تنفيذ حقيقي أصلاً** — `if PAPER_TRADING:` بدون `else`، والرصيد الحقيقي مكتوب `= 0` | نظام paper صريح ومعلن، بدون ادعاءات |
| رافعة x20 + مخاطرة تصل 7.5% لكل صفقة | رافعة x3 + مخاطرة 1% (سقف 2%) |
| محاكاة بدون رسوم ولا انزلاق سعري → أرباح وهمية | رسوم OKX Taker الفعلية (0.05%) + انزلاق 0.02% لكل تنفيذ |
| تجاهل الحد الأدنى لحجم عقود OKX | يجلب مواصفات العقود الحقيقية (`ctVal`/`lotSz`/`minSz`) ويرفض الصفقات المستحيلة بصدق |
| شموع دقيقة واحدة + دورة كل 5 ثوانٍ → الرسوم تلتهم الرصيد | شموع 5 دقائق + دورة كل 60 ثانية |
| عتبة إجماع 52% (= عملة معدنية تقريباً) | 65% مع 3 وكلاء نشطين على الأقل |
| لا محاكاة تصفية | تصفية معزولة تقريبية: الخسارة لا تتجاوز الهامش |
| إشارات على شموع غير مكتملة (إعادة رسم) | الإشارات على الشموع **المؤكدة** فقط |

## التشغيل

```bash
pip install requests   # الاعتماد الوحيد

python3 snowball_paper.py --once           # دورة واحدة ثم خروج
python3 snowball_paper.py --minutes 60     # ساعة كاملة
python3 snowball_paper.py                  # بلا توقف (Ctrl+C للإيقاف)
python3 snowball_paper.py --balance 2      # جرّب برصيدك الحقيقي وشاهد الواقع
```

## ماذا ستشاهد برصيد $2؟

الحد الأدنى لعقد BTC على OKX ≈ **$6 notional** — أي أن أغلب الإشارات سترفَض
برسالة صريحة. هذا ليس عيباً في النظام؛ هذا هو الواقع الذي سيواجهك على
المنصة الحقيقية أيضاً. العملات الوحيدة القابلة للتداول برصيد صغير هي ذات
عقود صغيرة جداً، والرسوم ستأكل نسبة كبيرة من كل صفقة.

## اختيار العملات (ديناميكي)

لا توجد قائمة ثابتة. عند كل تشغيل — وكل ساعة أثناء العمل — يفحص النظام
سوق OKX كاملاً (~400 زوج USDT-SWAP) ويختار:

- سيولة 24 ساعة **≥ $30 مليون** (قابلة للتعديل في `MIN_LIQUIDITY_USD`)
- **بدون BTC / ETH / SOL** (مستبعدة عمداً)
- **بدون الأصول المرمّزة غير الرقمية** (ذهب XAU، فضة XAG، أسهم مثل MSTR/MU)
- أعلى 20 عملة سيولةً من المتبقي
- العملة التي عليها صفقة مفتوحة تبقى تحت المراقبة حتى تُغلق صفقتها

## البنية

- **5 وكلاء** يصوّتون بأوزان: Trend (EMA 20/50/100)، Momentum (MACD)،
  RSI (ارتداد التشبع)، Volume (انفجار الحجم)، Bollinger (كسر النطاق)
- **إدارة مخاطر**: مخاطرة ديناميكية مع سلاسل الفوز/الخسارة،
  توقف يومي عند خسارة 5%، إيقاف كامل عند سحب 15% من القمة
- **إدارة صفقات**: SL = 1.5×ATR، TP = 3×ATR (عائد/مخاطرة 1:2)،
  نقل SL لنقطة الدخول بعد ربح 1×ATR، وقف متحرك 1.2×ATR
- **سجل دائم**: كل الصفقات ولقطات الرصيد في `snowball_paper.db` (SQLite)

## الحقيقة كاملة

لا يوجد نظام "خارق" يحوّل $10 إلى مليون — لا هذا ولا الأصلي ولا أي بوت آخر.
الغرض من هذا النظام هو **القياس الصادق**: شغّله أسابيع، وراقب هل الاستراتيجية
تربح فعلاً بعد الرسوم أم لا. إن كانت تخسر وهمياً، فستخسر حقيقياً. القرار
النهائي — إن وُجد — يكون بيدك أنت على المنصة مباشرة وعلى مسؤوليتك.
